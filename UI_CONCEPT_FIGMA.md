# 外卖热量追踪 App UI 概念图说明（Figma 版）

> 目标：在**不进入开发**的前提下，先完成可评审、可执行的 UI 概念设计蓝图。
> 使用方式：设计师可直接按本文件在 Figma 建立页面、组件与原型流。

## 1. 设计目标
- 一眼看懂：用户在 3 秒内知道“今天吃了多少、是否超标、下一步做什么”。
- 一步记录：拍照上传路径尽量缩短，减少跳转和输入负担。
- 行动导向：结果页优先展示“下一餐怎么吃”。
- 都市白领风格：简洁、克制、可信赖，不做过度花哨视觉。

## 3. 视觉基础（Foundation）

## 3.1 设计基线
- 设备基准：
  - iOS：390 × 844（iPhone 13/14）
  - Android：360 × 800
- 栅格：4pt 基础网格。
- 间距体系：4 / 8 / 12 / 16 / 20 / 24 / 32。
- 圆角：8（输入框）/ 12（卡片）/ 16（弹层）。

## 3.2 色彩（建议 Token）
- `Primary/500`：#2F6BFF（主行动色）
- `Success/500`：#16A34A（达标、正向）
- `Warning/500`：#F59E0B（注意）
- `Danger/500`：#EF4444（超标）
- `Neutral/900`：#111827（主文本）
- `Neutral/600`：#4B5563（次文本）
- `Neutral/200`：#E5E7EB（分割线）
- `Neutral/50`：#F9FAFB（页面底色）

## 3.3 字体层级
- 标题 H1：24 / 32，Semibold
- 标题 H2：20 / 28，Semibold
- 子标题：16 / 24，Medium
- 正文：14 / 22，Regular
- 辅助：12 / 18，Regular
- 数字强化（热量）：28 / 32，Bold

---

## 4. 组件库（Components）

## 4.1 核心组件
1. **Primary Button**
   - 状态：Default / Pressed / Disabled / Loading
2. **Meal Card（餐次卡片）**
   - 内容：时间、缩略图、热量区间、风险标签
3. **Calorie Ring（今日摄入环）**
   - 内容：已摄入/目标值、达标状态文案
4. **Risk Tag**
   - 类型：高油 / 高糖 / 高钠 / 低置信
5. **Advice Card**
   - 类型：下一餐建议 / 当日补救建议
6. **Trend Chart（7 日）**
   - 迷你柱图 + 超标点标识
7. **Bottom Tab Bar**
   - 记录 / 分析 / 我的

## 4.2 组件约束
- 卡片内边距统一 16。
- 同层级卡片间距 12。
- 高风险标签最多显示 2 个，其余合并为 `+N`。
- 建议卡片每次最多展示 3 条。

---

## 5. MVP 关键页面（Hi-Fi 概念）

## 5.1 首页（记录）
**目标**：让用户马上拍照、快速复盘今天状态。

### 区块结构（自上而下）
1. 顶部问候 + 日期
2. 今日摄入环（已摄入 / 目标）
3. 主按钮：`拍照记录一餐`
4. 最近记录（最多 3 条）
5. 空状态引导（无记录时）

### 关键文案
- 摄入环：`今日已摄入 1280 / 1800 kcal`
- CTA：`拍照识别热量`

---

## 5.2 上传页
**目标**：降低失败率，状态清晰。

### 状态
- Default：相机/相册双入口 + 餐次选择
- Uploading：进度条 + “识别中”提示
- Error：超时/格式错误/网络失败

### 错误文案示例
- `网络不稳定，识别超时，请重试`
- `仅支持 JPG / PNG，且小于 10MB`

---

## 5.3 结果页（核心）
**目标**：10 秒内让用户“理解结果并愿意执行建议”。

### 信息层级
1. 热量区间（大数字）
2. 风险标签（高油/高糖/高钠）
3. 热量来源拆解条
4. 建议卡（下一餐 + 今日补救）
5. 主按钮：`保存记录`
6. 次按钮：`换一组建议`（每日 3 次）

### 低置信状态
- 增加提示条：`识别置信度较低，建议补充：是否包含甜饮/额外酱料`。

---

## 5.4 分析页
**目标**：支持用户每周复盘，形成持续行为。

### 区块结构
1. 今日总结（是否达标）
2. 近 7 日趋势图
3. 高风险餐次列表
4. 一键纠偏建议

### 可视化建议
- 达标天：绿色柱
- 轻度超标：黄色柱
- 重度超标：红色柱

---

## 5.5 我的页
**目标**：管理目标与订阅入口。

### 区块结构
1. 用户资料卡（目标、体重、活动水平）
2. 目标设置入口
3. 订阅权益入口
4. 隐私与数据删除入口

---

## 6. 核心交互原型流（Prototype）

### Flow A：首次使用
欢迎页 → 填资料 → 首页空状态 → 拍照上传 → 结果页 → 保存记录 → 首页有记录状态

### Flow B：日常记录
首页 → 拍照上传 → 结果页 → 换建议（可选）→ 保存

### Flow C：复盘纠偏
分析页 → 点击超标日 → 进入详情 → 查看纠偏建议

---

## 8. 设计评审标准
- 任务可达性：核心任务（拍照记录）≤ 2 次点击可达。
- 可读性：关键数值与行动按钮在首屏可见。
- 一致性：组件与间距遵循 Token。
- 合规性：结果页与建议页均有“非医疗诊断”提示。

---

## 9. 下一步建议（仍不开发）
1. 先在 Figma 完成低保真线框评审（30 分钟）。
2. 再补高保真与动效原型（重点做结果页和分析页）。
3. 通过评审后再进入“PRD 页面级交互细化 + 技术评估”，最后才排期开发。
